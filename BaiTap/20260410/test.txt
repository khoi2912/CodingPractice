件名：偶数の合計が正しく計算されない不具合

概要：
1から指定した数値までの偶数の合計が正しく計算されない不具合が発生しています。

現象：
数値「10」を入力すると、結果が「1」と表示されます。

再現手順：
1. プログラムを実行する
2. 「10」を入力する
3. 表示された結果を確認する
4. 結果が「1」と表示されます。-> NG

実際の結果：
結果が「1」と表示されます。

期待結果：
結果が「30」と表示されることを期待します。

発生頻度：
毎回発生します。

原因：
偶数を検出した際にbreak文を使用しているため、ループが途中で終了しています。

影響範囲：
すべての入力値に対して、偶数の合計が正しく計算されません。

対応方法：
break文を削除し、偶数の場合のみsumに加算するように修正します。

優先度：
高



#include <stdio.h>

int main() {
    int n, i, sum = 0;

    printf("Enter a positive number: ");
    scanf("%d", &n);

    for (i = 1; i <= n; i++) {
        if (i % 2 != 0) {
            continue;
        }
        sum += i;
    }

    printf("Sum of even numbers from 1 to %d is: %d\n", n, sum);
    return 0;
}
